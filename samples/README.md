# Sample generations

Text generated by the trained Dante nanoGPT. These are produced **from scratch**
by a small character-level transformer — the model has only ever seen the raw
characters of the *Divine Comedy* and learned Italian spelling, Dante's
vocabulary, and the rhythm of the *terzine* purely from next-character
prediction.

## Run 1 — first full Colab run (T4 GPU)

**Date:** 2026-06-15

**Hyperparameters**

| param | value |
|-------|-------|
| n_layer | 6 |
| n_head | 6 |
| n_embd | 384 |
| block_size | 256 |
| batch_size | 64 |
| max_iters | 5000 |
| learning_rate | 3e-4 |
| dropout | 0.1 |
| precision | float16 (T4) |
| params | ~10.7M |

**Results**

- **Best validation loss: 1.3538** (this is the checkpoint that is saved and used
  for sampling).
- Final-step train loss ~0.14 with val loss ~2.33 → the model **overfits** the
  small corpus by the end of training (it nearly memorizes the train split).
  Because `train.py` always keeps the *best* checkpoint by validation loss, the
  saved model is the earlier, better-generalizing one, not the overfit final
  state.

## Observations & ideas for a better run

The corpus is small (~hundreds of thousands of characters), so a 10.7M-parameter
model overfits quickly. To improve generalization next time, try one or more of:

- Increase `--dropout` (e.g. 0.2–0.3).
- Use a smaller model (`--n_layer=4 --n_embd=256`) and/or fewer iterations.
- Rely on the best-checkpoint behavior (already implemented) — the saved model is
  taken from the lowest validation loss, not the final step.
